The goal of `testMuBrowser` is to provide a robust, open-standard compatibility layer between Agentic AI frameworks and LambdaTest (or local) browser infrastructure.

## How Can I Contribute?

### Reporting Bugs

This section guides you through submitting a bug report.
*   **Use the Issue Template:** Please use the provided bug report template in `.github/ISSUE_TEMPLATE/`.
*   **Provide Reproduction Steps:** A small script that reproduces the crash or unexpected behavior is incredibly helpful.

### Suggesting Enhancements

*   **Open an Issue first:** Discuss your idea before writing code. We want to ensure it aligns with the project roadmap (especially regarding Steel.dev parity).

### Pull Requests

1.  **Fork the repo** and create your branch from `main`.
2.  **Install dependencies**:
    ```bash
    npm install
    ```
3.  **Run Tests**:
    ```bash
    npm test
    ```
4.  **Linting**: Ensure your code follows the existing style (TypeScript).
5.  **Commit**: Use descriptive commit messages.

## Development Setup

The project is written in TypeScript.

*   `src/testMuBrowser/`: Main SDK source code.
*   `examples/`: Usage demos.
*   `docs/`: Documentation.

To build the project:
```bash
npm run build
```
